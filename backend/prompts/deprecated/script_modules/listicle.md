Listicle structure (script stage). Write exactly the items the outline planned, one section each, in order, each as a turn.

- **Hook names no items.** Deliver the title's promise, give the count and the stakes, roll into section 1 - keep it short and engaging. Naming any item spoils its section.

- **Open each section on the turn, not the item.** Start with what's surprising about THIS one - the belief that's wrong, the misconception they've been fed, the failure they've probably lived, or the want they've carried.

- **One promise, many proofs.** Each item's landing is the title's promise coming true again - different fact, same currency. No item sends the viewer out to act; only the conclusion does, on the peak item. If the script starts reading like separate pitches, the items have stopped landing on the promise.

- **Land the feeling.** Each item has one job - one thing to make the viewer feel or picture (the outline names it). Build the section toward that, and let the strongest-feeling items (the peaks) run longer.

- **Stitch the list together.** When a later item touches an earlier one's ground - the same rule inverted, the same tool, the same spot in the yard - say so in a clause. Callbacks and cross-references are what make a list feel like one video instead of islands.

- **Vary how sections open.** Scene, myth-bust, confession, the rival who succeeds. If they all open the same way the list goes flat. Look at the sample for how it switches shape item to item.

- **Sections can just end.** Don't force a teaser bridge into the next one - a supporting item can simply stop, and the next begins.

- **Segment title is the literal item name.** Copy it verbatim from `outline.sections[i].title` into `segments[i].title`. Don't rewrite it into a punchier headline — downstream stages (visuals, assembler) match on this exact string. The surprise belongs in the opening of `narration`, not in the title.
